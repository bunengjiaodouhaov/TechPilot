# TechPilot 产品基线

## 一句话定义

TechPilot 是一个面向开发者的技术调研与代码理解平台，通过文档 RAG、Code RAG 和 Agent，帮助用户检索技术证据、理解开源仓库，并把分析结果转化为可执行的项目改造任务。

## 三个核心能力

### 1. 技术文档调研

导入官方文档、论文或技术资料，完成检索、可信问答、引用溯源和技术方案对比。

### 2. 开源仓库理解

分析代码仓库的目录、文件、类、函数、依赖和调用关系，并基于代码证据回答问题。

### 3. 能力差距分析

读取目标岗位 JD，从当前项目中寻找能力证据，识别能力缺口，并生成可执行的项目改造任务。

## 第一版主业务闭环

导入技术资料或代码仓库 → 提出问题 → 检索文档与代码证据 → 生成带引用的分析结果 → 识别项目或能力缺口 → 生成改造任务 → 人工确认并执行。

## 第一版禁止功能

- 不做生活助手、天气、日程、记账或泛聊天。
- 不把学习计划和面试题生成作为核心模块。
- 不做通用低代码工作流平台。
- 不复刻 Dify、RAGFlow 或 Open WebUI。
- 不设计多个职责不清晰的 Agent。
- 不在 RAG 和 Code RAG 完成评测前美化 UI。
- 不在第一版实现多模态 OCR、全仓库代码生成或自动修改代码。
- 不优先进行 LoRA 微调；模型微调只作为后期加分实验。
- 不能进入主业务闭环的功能暂不开发。

## 范围变更规则

所有新增需求先进入 backlog。只有能够增强技术调研、Code RAG 或能力分析主线，并且具备明确验收与评测方法时，才允许进入正式迭代。
